# Спецификация наблюдаемости (observability)

## Текущее состояние
- Вход в `process_message` (user_id, длина промпта)
- Вызов `search_internet` (запрос, результат: количество результатов или ошибка)
- Запрос к Mistral (модель, количество токенов, время ответа)
- Исключения (с traceback)

##Метрики (для Prometheus)
- `mistral_requests_total` (label: status=success/error)
- `search_requests_total` (label: status=success/error)
- `search_results_count` (гистограмма)
- `dialog_turn_duration_seconds` (гистограмма)

## Проверки качества (автоматические)
- **Детекция галлюцинаций**: сравнивать ссылки в ответе агента с реальными ссылками из результатов поиска.
- **Полнота**: проверять, что агент использовал инструмент хотя бы один раз, если это нужно.
- **Завершённость**: ответ не должен обрываться посередине (в коде есть разбивка на чанки по 4000 символов для Telegram, но это не влияет на логику агента).
